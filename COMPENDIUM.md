## 1. System Architecture & Principles

### 1.1. The Dual-Layered Brain

This isn't a static note-taking setup — it's a **living digital nervous system**.
The BROski-Obsidian-Brain v3.0 architecture splits the load between two critical layers.

| Layer | What It Is |
|---|---|
| **The Second Brain** (`HYPERFOCUS_ZONE/`) | Local Obsidian vault. PARA scaffold. Data substrate + sensory environment. |
| **The Hyper Brain** (Container #30, Port 8100) | FastAPI engine. 12 API endpoints. 8 core AI modules. The heartbeat. |

**The Connectivity Limbs** — three bridges between the Docker metropolis and your vault:
- **GitHub Webhook Server** — real-time sync
- **HyperAgent MCP Bridge** — connects to technical SDKs
- **Focus Tracker / Analytics** — writes productivity snapshots back to vault

### 1.2. Neurodivergent-First Sacred Design Rules

> We build for the ADHD, Autistic, and Dyslexic brain. No fluff, just results.

1. **Productivity is the Soul of the Pet** — Metadata (XP, energy) only shifts in response to real-world labour processed by the engine.
2. **Fast Feedback** — Immediate dopamine-friendly rewards are mandatory to crush task paralysis.
3. **No Fluff Interactivity** — Every interaction must provide utility, not just maintenance overhead.
4. **Absolute Ecosystem Sync** — If the connection between the pet and the vault is severed, the pet loses its brain. Ride or die.
5. **Sensory Integrity** — Level 12 CSS eliminates visual noise and prevents executive dysfunction.

### 1.3. The Docker Metropolis (HyperCode-V2.4)

**32-container Docker metropolis.** Pay attention to the Failure Chain:

- Any container fails → lost a **limb** (coding stats, specific pet update)
- **Container #30 fails → total brain shutdown** 🚨
  - Vault reverts to static notes
  - All AI-driven functionality goes dark: briefings, filtering, splitting

> Container #30 (Hyper Brain) is the single point of failure. Protect it.

### 1.4. Sensory Architecture (HyperFocus CSS — Level 12)

Implemented via `hyper-brain-themes.css`:

| User Condition | Visual Adjustments | Cognitive Benefit |
|---|---|---|
| ADHD | High-focus palettes, reduced clutter, minimised distractions | Bypasses interface friction, facilitates hyperfocus entry |
| Autism | Specialised typography, sensory-managed colour schemes | Mitigates sensory overload, stable calm workspace |
| Dyslexia | Optimised typography for legibility, high-contrast spacing | Enhances info processing, accessible vault navigation |
